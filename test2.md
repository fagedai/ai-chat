# AI助手API接入与高级功能指南 v1.5

## API接入流程

1. 在内部平台申请 API Key，格式为 `sk-internal-{部门代号}-{随机串}`
2. 调用基础地址：`https://ai-api.company-internal.com/v2`
3. 请求头需携带 `Authorization: Bearer {API Key}` 和 `X-Device-ID: {设备标识}`
4. 支持流式（SSE）和非流式两种响应模式

## 速率限制

| 用户级别 | 每分钟请求数 | 并发数 | 单次最大Token |
|----------|-------------|--------|--------------|
| 普通员工 | 20 | 3 | 4096 |
| 部门经理 | 60 | 10 | 8192 |
| VP及以上 | 200 | 30 | 32768 |

超出速率限制会返回 HTTP 429 状态码，响应体中包含 `retry_after` 字段（单位：秒）。

## 高级功能

### 自定义知识库
- 支持上传 PDF、Word、Markdown、TXT 格式文档
- 单个知识库最大 500MB，最多 1000 个文件
- 向量化采用 BGE-M3 模型，支持中英文混合检索
- 检索方式：向量相似度 + BM25 关键词混合排序

### 批量处理
- 批量问答接口：`POST /v2/batch/chat`
- 单次最多 50 条问题，处理结果异步回调
- 批量任务保留 7 天，过期自动清理

### 模型选择
- `gpt-internal-4o`：通用对话，响应快，适合日常问答
- `gpt-internal-4o-mini`：轻量模型，成本低，适合简单分类任务
- `deepseek-internal-v3`：擅长代码生成和逻辑推理
- 所有模型均支持 function calling 和 JSON 模式输出

## 故障排查

### 常见HTTP状态码
- `200`：成功
- `401`：API Key 无效或已过期
- `403`：权限不足，例如普通员工调用 VP 专属接口
- `429`：超出速率限制
- `500`：服务端内部错误，请联系技术支持

### API层错误码
| 错误码 | 含义 | 处理方式 |
|--------|------|----------|
| API-2001 | API Key 格式错误 | 检查 Key 是否符合 `sk-internal-` 前缀规范 |
| API-2002 | 设备未授权 | 确认设备已在管理后台绑定 |
| API-2003 | 请求体超过大小限制 | 单次请求体不得超过 1MB |
| API-2004 | 模型不可用 | 检查模型名称或联系管理员开通权限 |
