# 系统错误码参考手册 v3.2

## 1. 认证与授权模块

### E-1001：Token 过期
- 触发条件：JWT Token 超过有效期（默认 24 小时）
- HTTP 状态码：401 Unauthorized
- 解决方案：调用 `/api/auth/refresh` 刷新 Token，若 Refresh Token 也过期则需重新登录
- 影响范围：所有需要认证的 API 接口
- 常见误操作：前端未在拦截器中自动刷新 Token，导致用户被强制登出

### E-1002：权限不足
- 触发条件：用户角色不具备当前操作的权限
- HTTP 状态码：403 Forbidden
- 解决方案：联系管理员分配对应角色权限，或检查前端是否传入了正确的 role_id
- 影响范围：管理后台的所有写操作（增删改）
- 关联错误：常与 E-1003 同时出现，当角色被删除但缓存未清除时

### E-1003：角色不存在
- 触发条件：请求中携带的 role_id 在系统中不存在
- HTTP 状态码：404 Not Found
- 解决方案：确认角色列表（GET /api/roles），使用有效的 role_id
- 排查步骤：
  1. 检查数据库中 roles 表是否有对应记录
  2. 确认 Redis 缓存是否与数据库同步
  3. 若缓存不同步，执行 `cache:clear --scope=roles` 命令

### E-1004：OAuth 回调失败
- 触发条件：第三方 OAuth（微信、GitHub）回调时 state 参数不匹配
- HTTP 状态码：400 Bad Request
- 解决方案：
  - 检查 OAuth 应用的回调地址配置是否正确
  - 确认 session 中存储的 state 值未被清除
  - 微信 OAuth 需要额外配置 app_secret 的加密方式

## 2. 数据处理模块

### E-2001：数据格式校验失败
- 触发条件：请求体不符合 JSON Schema 定义
- HTTP 状态码：422 Unprocessable Entity
- 解决方案：参照 API 文档中的请求体 Schema，修正字段类型或必填项
- 典型场景：
  - 日期字段传入了非 ISO 8601 格式（如 "2024/01/01" 应为 "2024-01-01"）
  - 金额字段传入了字符串而非数字类型
  - 数组类型字段传入了单个对象

### E-2002：批量操作超限
- 触发条件：单次批量请求超过 1000 条记录
- HTTP 状态码：413 Payload Too Large
- 解决方案：将批量请求拆分为多次，每次不超过 1000 条
- 性能建议：
  - 建议每批 500 条，兼顾效率与稳定性
  - 批量删除操作建议先查询确认，避免误删
  - 大批量导入推荐使用异步任务接口 POST /api/tasks/batch-import

### E-2003：并发写入冲突
- 触发条件：两个请求同时修改同一条记录（乐观锁检测）
- HTTP 状态码：409 Conflict
- 解决方案：获取最新数据后重新提交，请求中需携带最新的 version 字段
- 技术原理：系统使用乐观锁机制，每条记录包含 version 字段，写入时比较版本号
- 重试策略：建议最多重试 3 次，间隔 100ms、200ms、400ms（指数退避）

## 3. 文件存储模块

### E-3001：文件类型不支持
- 触发条件：上传了不在白名单中的文件类型
- 支持的类型：jpg, jpeg, png, gif, webp, pdf, docx, xlsx, csv, txt, md
- HTTP 状态码：415 Unsupported Media Type
- 解决方案：将文件转换为支持的格式后重新上传

### E-3002：存储空间不足
- 触发条件：当前租户的存储配额已用尽
- HTTP 状态码：507 Insufficient Storage
- 解决方案：清理不需要的文件或联系管理员扩容
- 配额说明：
  - 免费版：500MB
  - 基础版：10GB
  - 专业版：100GB
  - 企业版：无限制

### E-3003：文件处理超时
- 触发条件：文件解析（如 PDF 提取、图片压缩）超过 30 秒
- HTTP 状态码：504 Gateway Timeout
- 解决方案：
  - 减小文件大小或降低分辨率
  - 使用异步处理接口 POST /api/files/async-process
  - PDF 文件建议先拆分为多个小文件

## 4. 系统级错误

### E-9001：数据库连接池耗尽
- 触发条件：并发请求超过连接池上限（默认 20 个连接）
- 影响：所有数据库操作都会排队等待
- 解决方案：
  - 短期：重启服务释放连接
  - 长期：优化慢查询，增加连接池大小（DB_POOL_SIZE 环境变量）
  - 监控：关注 pg_stat_activity 中的 idle in transaction 状态连接

### E-9002：Redis 缓存失效
- 触发条件：Redis 服务不可用或连接超时
- 影响：系统降级为直接读数据库，响应时间增加约 3-5 倍
- 解决方案：
  - 检查 Redis 服务状态：`redis-cli ping`
  - 检查网络连通性：`telnet redis-host 6379`
  - 启用本地缓存兜底：设置 FALLBACK_CACHE=true

### E-9999：未知系统错误
- 触发条件：未预期的异常
- HTTP 状态码：500 Internal Server Error
- 解决方案：收集 request_id 提交工单，技术团队将根据日志排查
